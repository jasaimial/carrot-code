# Journal — 2026-05-14 — Bootstrap

First session. No code yet; goal was to set the project up properly.

## Context

Starting state: an empty `carrot-code/` folder containing a single file, `docs/starter.txt` — a project charter generated by an external chat assistant for a fan-game version of the *Press Start!* book series. User wanted to take it further with a different agent (GitHub Copilot in VS Code), and explicitly framed the project as a **dual-purpose learning vehicle**: game programming + Spec Kit / SDD practice.

## What happened

1. **Reviewed the starter charter critically.** Identified seven gaps — most importantly that it ignored the spec-kit angle entirely, prescribed a multi-agent workflow that adds coordination cost without learning value, and used copyrighted IP on a public repo.
2. **Asked four gating questions** (IP handling, learning priorities, engine choice, MVP scope) instead of guessing.
3. **Locked the architecture commitments** based on the answers — production-quality from day one, data-driven extensibility (Tiled JSON levels, typed entity configs), TypeScript + Phaser 3, Vitest, PWA + CI.
4. **Confirmed prerequisites** already on the machine: git, gh, node, npm. Missing: `uv`.
5. **Installed `uv`** via winget. Hit the standard "binary not on PATH" issue, fixed with `uv tool update-shell` + manual session PATH.
6. **Initialized git on `main`** with a curated `.gitignore` and a baseline commit *before* invoking spec-kit (so we control the initial commit, not the tool).
7. **Installed `specify-cli` v0.8.10** pinned via `uv tool install … @v0.8.10`.
8. **Ran `specify init --here --integration copilot --script ps --force --ignore-agent-tools`.** Scaffolding landed cleanly: `.specify/`, `.github/prompts/`, `.github/agents/`, `copilot-instructions.md`, plus a benign `.vscode/settings.json` patch.
9. **Committed the scaffold** as a separate commit so the diff is reviewable in isolation.
10. **Wrote this learning-docs structure** (`docs/learning/` with index, setup how-to, workflow reference, this journal entry, and a template for future entries).

## Decisions

| Decision | Rationale | Reversible? |
|---|---|---|
| Original IP, not fan-game | Public-repo-safe; lets us deploy + share without legal risk | Hard — would require renaming/replacing all assets |
| Phaser 3 + TypeScript + Vite | MIT-licensed, mature, Tiled support enables data-driven levels; TS for production-quality | Medium — engine swap mid-project is painful but doable |
| Vertical slice first (one polished level) | User explicitly wants production-quality, not breadth-first throwaway code | Easy |
| Tiled JSON levels via generic `LevelScene` | Enforces "new level = drop in JSON, no scene code edits" extensibility | Medium |
| Kenney.nl CC0 placeholder art for v0 | Unblocks gameplay work; pixel-art pipeline is a separate later track | Easy |
| Spec-kit pinned to v0.8.10 | Reproducibility; v0.10 has breaking flag changes | Easy |
| PowerShell scripts (`--script ps`) | Native to user's environment; no WSL needed | Hard once specs reference scripts |
| Defer GitHub remote until after constitution + first spec | Better initial push (more meaningful baseline) | N/A |

## What worked

- **Asking gating questions before building anything.** Four questions, ~30 seconds each, prevented hours of rework.
- **Critiquing the existing charter explicitly** instead of silently replacing it. Made the *why* of every change visible.
- **Pinning the spec-kit version.** Avoids "it worked yesterday" surprises.
- **Two-commit bootstrap** (baseline, then scaffold) — clean, reviewable history.

## What didn't

- **`specify --help` crashes** in captured-output environments (cosmetic). Worked around by reading docs + using `specify <subcommand> --help`.
- **First instinct was `--ai copilot`** — that flag is deprecated; correct one is `--integration copilot`. Saved by reading help output before running.
- **PATH issue post-`uv tool install`** — predictable but still cost a few minutes. Now documented in [00-setup-from-zero.md](../00-setup-from-zero.md#4-install-the-specify-cli).

## What I'd do differently

- Read `specify init --help` *first* next time, before drafting the command. Would have caught `--ai` deprecation immediately.
- Could have skipped `--ignore-agent-tools` and let it tell me what it was looking for — would have been informative.
