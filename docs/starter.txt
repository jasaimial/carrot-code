📋 Project Charter: Super Rabbit Run (PWA)
1. Storyline & Game Logic
Goal: Translate the "meta" book experience into a playable 2D platformer.
The "Sunny" Layer: Implement a UI overlay that looks like a handheld console screen. Occasionally, a text box from "Sunny" appears to give hints (Tutorial mode).
The Chapter System: Level 1 (The Forest), Level 2 (The Robot Factory), Level 3 (King Viking’s Castle).
Mechanics:
Health: 3 Hearts (Classic book style).
Power-ups: The "Super Carrot" turns the rabbit into "Super Rabbit Boy" (invincibility/speed).
Game Over: A pixelated screen that asks to "Insert Coin" (or just click to restart).
________________________________________
2. Visuals & Assets (The Look)
Goal: Replicate Thomas Flintham’s 8-bit aesthetic.
Tooling:
Sprite AI: Best for generating game-ready 32x32 pixel sprites of rabbits and robots.
Aseprite: (Manual) The industry standard for cleaning up AI-generated pixels and creating "Walk" cycles.
Environment:
Parallax Scrolling: Background clouds move slower than the foreground to give depth.
Tilemaps: Use 16x16 pixel blocks for the ground to keep that "blocky" retro feel.
________________________________________
3. Engineering & Infra (The "How")
Goal: A high-performance web game that works offline.
Game Engine: Phaser 3 (JavaScript/TypeScript). It is the most robust engine for 2D web games in 2026.
PWA Features:
manifest.json: For "Add to Home Screen" support.
service-worker.js: To cache the 8-bit music and sprites for offline play.
Deployment: GitHub Pages or Netlify (Free, fast, and supports HTTPS required for PWAs).
________________________________________
4. AI Agent Workflow (Your Virtual Team)
To build this, you should use a "Multi-Agent" approach rather than just one chat box.
A. The "Architect" (Me / Gemini)
My Role: I am your Project Manager and Logic Lead. I can write the full Phaser 3 code, design the math for the jumping physics, and troubleshoot bugs.
The Gap: I cannot "see" your local files in real-time or click "Play" to test the game for you. I also cannot generate the actual .png files directly into your folder.
B. The "Builder" (Cursor / Replit Agent)
Recommended Tool: Cursor (IDE) or Replit Agent.
Why: These are "Agentic IDEs." You can tell them "Add a double-jump mechanic to Super Rabbit Boy," and they will actually edit your files, install the necessary libraries, and start the local server.
The Gap: They can be "literal." They might give you a generic jump when you need a "Rabbit Boy" jump. You use me (Gemini) to refine the "feel," then paste my logic into them.
C. The "Artist" (Sprite-AI / Midjourney)
Why: Specialized for the 8-bit aesthetic.
The Gap: AI often struggles with "consistency" (making the rabbit look the same in every frame). You will need to use a tool like Aseprite to manually fix the eyes or ears of the rabbit.  
________________________________________
5. Implementation Roadmap (Phase 1)
Scaffolding: Use a Phaser 3 + Vite starter template.
The "Hero" Sprite: Create a simple 32x32 pixel rabbit.
The Floor: Create a "Solid" group in Phaser so the rabbit doesn't fall through the bottom of the screen.
tGravity: Set arcade.physics.gravity.y = 800.
The PWA Prompt: Add the code to detect if the user is on mobile and show an "Install App" button.
